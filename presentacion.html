2. Tecnologías Utilizadas

HTML5

La base de la web, proporcionando una estructura clara y accesible para todos los dispositivos.

CSS3

Encargado del diseño visual, los colores, y las animaciones que hacen atractiva la página.

Bootstrap

Herramienta clave para conseguir que la página se adapte automáticamente a las pantallas de los móviles.

JavaScript

El "cerebro" que añade dinamismo, como mostrar elementos al hacer scroll o validar el formulario.

Para el desarrollo, he apostado por tecnologías del lado del cliente puro.
(Click) HTML5 para un buen etiquetado.
(Click) CSS3 muy avanzado, usando técnicas de diseño moderno como el Glassmorphism o cristales esmerilados.
(Click) Bootstrap 5 como corazón del sistema de rejilla móvil.
(Click) Y JavaScript nativo, sin jQuery, para que la web cargue como un rayo.

3. Retos: Interfaz de Usuario

Diseño 100% Responsivo

El Reto: Adaptar una web con múltiples columnas y elementos gráficos para que se viera perfecta en un teléfono móvil.

La Solución: Reorganización automática de contenidos en bloques apilados mediante CSS y Bootstrap.

Navegación Amigable

El Reto: Lograr una experiencia de usuario (UX) fluida donde los menús y la información siempre estén a mano sin molestar.

La Solución: Menú superior fijo que compensa su altura automáticamente para no tapar los títulos.

Llegar a ese resultado final tuvo sus retos técnicos.
(Click) Al fijar el menú, tapaba la página. Lo solucioné con paddings dinámicos.
(Click) Pero el mayor orgullo es el rendimiento. En vez de usar librerías pesadas, usé la API IntersectionObserver para que los elementos detecten cuando entran en pantalla.

4. Retos Técnicos: Tablas y Lógica

Tablas Responsivas

Problema: Un calendario de 7 columnas explota en móviles.

Solución: Modificación nativa de CSS para encoger fuentes y apilar celdas formando "tarjetas" verticales.

Formulario sin Backend

Problema: Sin servidor, no había correos reales.

Solución: Script en JS que intercepta el envío, comprueba campos y simula el éxito limpiando todo.

Otros dos grandes retos:
(Click) El diseño adaptativo de tablas gigantes. En móviles, mediante CSS puro, forzamos que las tablas se conviertan en bloques apilados o reduzcan su tamaño.
(Click) Y por último, el formulario, que carece de servidor, está validado 100% por cliente con JavaScript interceptando el envío.

5. Traducción e Inteligencia Artificial

La página web cuenta con una versión completa en inglés para captar público internacional. Para ello, nos apoyamos en la I.A.

Precisión en el Código

La IA permite traducir los textos integrados en el código sin romper accidentalmente las etiquetas HTML o los estilos.

Terminología "Geek"

Los traductores tradicionales fallan con juegos de rol y cartas. La IA comprendió el contexto para usar términos exactos de la afición.

Además, exigimos que la web estuviese en dos idiomas.
(Click) En lugar de Google Translate, utilicé IA generativa para procesar los archivos HTML completos.
(Click) Usé técnicas de "Prompting" para que la IA actuase como un friki y tradujese términos complejos con naturalidad.
(Click) Y como el diseño usa Flexbox, los diferentes largos de los textos se acomodaron solos en las cajas, sin romperse.

5. Translation and Artificial Intelligence

The website features a complete English version to attract international audiences. To achieve this, we rely on A.I.

Code Precision

AI allows translating embedded text within the code without accidentally breaking HTML tags or styles.

"Geek" Terminology

Traditional translators fail with role-playing and card games. The AI understood the context to use the exact hobby terms.
