Introducción y Objetivos

Creación de un escaparate digital dinámico para atraer al público al local físico.

Promoción

Visibilidad de productos, eventos y torneos locales.

Conectividad

Facilitar el contacto directo y la localización física.

Objetivos Técnicos

Accesibilidad

Diseño 100% Responsive e internacionalizado (Inglés/Español).

Diferenciación

Reforzar la marca visual y destacar el valor humano del equipo.

La Empresa: Plutón Games

Punto de encuentro de referencia en Vigo para el ocio alternativo.

  • Catálogo: Juegos de mesa, TCG, manga y literatura fantástica.
  • Ventaja competitiva: Asesoramiento especializado humano.

Tecnologías Utilizadas

HTML5

Etiquetado semántico puro para mejorar SEO y accesibilidad estructural.

CSS3 Moderno

Variables nativas, flexbox, y efectos avanzados de Glassmorphism.

Bootstrap v5.3

Sistema de Grid responsive y componentes modulares UI.

JavaScript

Lógica de cliente nativa: Intersection Observer y validaciones.

Proceso de Desarrollo

  1. Planificación: Esqueleto HTML y enrutado base.
  2. Maquetación: Grid de Bootstrap y diseño por bloques (Cards).
  3. Personalización: Creación de `style.css` inyectando la identidad visual corporativa.
  4. Interactividad: Implementación de Javascript para animaciones.
  5. Internacionalización: Duplicación y traducción total al inglés.

Retos Técnicos: Soluciones

1. Superposición del menú fijo

Problema: El menú `fixed-top` ocultaba el contenido principal.
Solución: Implementación de clases de relleno dinámico para compensar la altura del menú.

2. Animaciones de rendimiento

Problema: Animar el scroll con métodos antiguos bloqueaba la interfaz.
Solución: Uso de la API Intersection Observer nativa de JavaScript, añadiendo clases CSS de forma optimizada.

Retos Técnicos: Soluciones

3. Formulario sin Backend

Problema: Imposibilidad de envío de emails al ser web estática.
Solución: Script JS propio que intercepta el envío, valida usando HTML5 y simula el éxito limpiando los campos visualmente.

4. Legibilidad sobre patrones

Problema: El fondo dinámico dificultaba leer el texto.
Solución: Arquitectura visual basada en componentes tipo "tarjeta" limpias (Glassmorphism y fondos sólidos).

Traducción con Inteligencia Artificial

Herramienta

Se utilizó Google Gemini como motor de traducción mediante Prompt Engineering.

Reto

El vocabulario "Geek". Traducir "Juegos de cartas coleccionables" requería contexto del sector.

La Solución

  • IA contextualizada al sector.
  • Respeto estricto del árbol DOM y clases CSS originales.
  • Flexbox resolvió los desajustes de longitud de palabras.

Escalabilidad y Futuro

Vías de crecimiento sobre el Frontend estático desarrollado:
